## 5. API 辅证结果（摘要）

详见 `api-check-results.json`，关键结论如下：

### 5.1 通过项

- 登录成功：`POST /api/v1/auth/token` -> `200 SUCCESS`
- CSRF 获取成功：`GET /api/v1/secure/antiforgery` -> `200 SUCCESS`
- 运行态任务（PascalCase 参数）可用：`GET /api/v1/runtime/tasks?PageIndex=1&PageSize=20&SortDesc=false` -> `200 SUCCESS`
- 未认证访问运行态被拒：`GET /api/v1/runtime/apps/.../schema`（无 token）-> `401 UNAUTHORIZED`
- 写接口缺失幂等键被拦截：`POST /runtime/.../records` -> `400 IDEMPOTENCY_REQUIRED`
- 跨租户访问（低代码详情）被拒绝：`GET /api/v1/lowcode-apps/1`（错租户头）-> `403 FORBIDDEN`

### 5.2 失败/异常项

- 低代码应用创建失败：`POST /api/v1/lowcode-apps` -> `500 SERVER_ERROR`
- 运行态目标应用不存在：
  - `GET /api/v1/runtime/apps/crm_demo_2026/pages/customer-form/schema` -> `404 NOT_FOUND`
  - 对应菜单返回空 items
- 查询参数兼容性差异：
  - `pageIndex/pageSize`（camelCase） -> `400` 校验失败
  - `PageIndex/PageSize`（PascalCase） -> `200`
- 跨租户头在 `GET /api/v1/platform/overview` 未触发拒绝（仍 `200 SUCCESS`），与“Tenant claim/header 强一致拒绝”预期存在偏差。

## 6. DoD 六维度验收结论

| 维度 | 结论 | 依据 |
|---|---|---|
| 标准统一 | 通过 | 运行态入口与统一布局可见，文档基线已落地 |
| 运行时闭环 | 有条件通过 | 运行态框架可访问，但当前示例 app/page 不存在，未形成有效业务页面闭环 |
| 数据闭环 | 不通过 | 低代码应用创建接口 `500`，导致“创建→发布→访问→提交”链路无法完成 |
| 治理与安全 | 有条件通过 | 幂等、鉴权、部分租户边界有效；但平台总览跨租户头未被拒绝 |
| 交付工程化 | 通过（本轮未重跑生成） | GUI 与 API 测试资产可复用，未触碰业务代码 |
| 运维可观测 | 有条件通过 | 关键请求可记录并回放，仍存在历史登录日志写入告警（`LoginLog.Message` 非空约束） |

## 7. 最终结论

**Gate-R2 结论：有条件通过。**

当前平台控制面、应用工作台框架、运行态布局与治理入口均可访问，安全基线（认证/幂等）大体有效；但由于低代码应用创建接口 `500`，以及运行态示例应用数据缺失，尚不能证明“设计 -> 发布 -> 访问 -> 提交数据”完整闭环已恢复到可交付状态。

## 8. 建议优先修复项

1. 修复 `POST /api/v1/lowcode-apps` 的 `500 SERVER_ERROR`（P0）。
2. 补齐至少一个可运行示例应用（含发布页面），用于运行态闭环验收（P0）。
3. 统一分页查询参数兼容策略（camelCase/PascalCase）并更新契约示例（P1）。
4. 复核 `platform/overview` 的租户一致性校验是否满足基线要求（P1）。
