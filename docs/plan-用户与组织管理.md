# 用户与组织管理实施计划（前端 / 后端）

> 适用范围：员工管理、部门管理、职位管理、租户管理、应用管理、应用数据源管理
> 目标：实现系统用户与组织的全生命周期管理，保证权限配置一致、可控、可追溯

---

## 一、总体原则

- 分层结构：严格遵循 Clean Architecture，Domain → Application → Infrastructure → Presentation。
- 强类型约束：后端 DTO/实体/配置全强类型；前端 TypeScript 全量类型标注。
- 异步优先：所有 Controller/Service/Repository 使用 async/await。
- 租户隔离：数据访问必须带租户边界，跨租户默认不可见。
- 审计可追溯：关键操作必须记录审计日志并支持查询。
- 契约同步：所有接口变更需同步更新 `docs/contracts.md`。

---

## 二、前端计划（Vue 3 + TS + Ant Design Vue）

### 2.1 信息架构与路由

- 建立 6 个模块入口：员工、部门、职位、租户、应用、应用数据源。
- 路由与菜单权限对齐，基于角色/权限控制可见性。

### 2.2 通用组件与能力

- 列表页：分页、筛选、排序、批量操作。
- 表单弹窗：新增/编辑/启停、校验提示。
- 组织树：树形展示与拖拽排序（如需）。
- 导入导出：支持模板下载、导入校验、导出过滤。

### 2.3 模块功能

**员工管理**
- 基础信息维护（姓名、工号、联系方式等）。
- 账号创建与启用/停用、状态（在职/离职/冻结）。
- 登录标识管理（用户名/手机号/邮箱）。
- 密码重置与安全策略提示（首次登录改密等）。
- 角色与权限分配。
- 员工与部门/职位绑定。
- 审计入口（操作记录与追溯）。

**部门管理**
- 树形组织新增、编辑、启停、排序。
- 上下级关系调整与组织编码维护。
- 统计查询（按组织维度）。

**职位管理**
- 职位新增、编辑、启停、排序、分类。
- 职位与员工/部门关联（主岗/兼岗可选）。
- 权限模板推荐/授权入口（可选）。

**租户管理**
- 租户创建、信息维护、启停。
- 租户管理员设置。
- 配额/资源管理（可选）。
- 租户内组织与用户入口。

**应用管理**
- 应用新增/编辑/启停、标识维护。
- 版本与发布状态（可选）。
- 配置项维护（功能开关/参数等）。
- 租户开通/变更/停用与审计。

**应用数据源管理**
- 数据源新增/编辑/启停。
- 连接测试与权限校验提示。
- 应用绑定关系维护。
- 变更记录与审计入口。

### 2.4 前端类型与契约

- 所有 API 请求/响应使用强类型。
- 同步更新 `docs/contracts.md`。

## 三、后端计划（.NET 10 + Clean Architecture + SqlSugar + SQLite）

### 3.1 领域建模（Domain）

- 员工、部门、职位、租户、应用、数据源、角色、权限、审计日志等实体。
- 明确聚合与关系：员工-部门-职位、租户-组织-用户、应用-数据源-租户授权。

### 3.2 应用层（Application）

- DTO 定义与验证（新增/编辑/启停/查询/批量）。
- 映射配置与统一响应模型。
- 业务用例编排，确保权限与状态流转一致。

### 3.3 基础设施层（Infrastructure）

- SqlSugar 仓储实现，所有数据访问走 Repository。
- 租户上下文注入与隔离策略。
- 数据源连接信息加密存储与脱敏展示。
- 审计日志落库与检索。

### 3.4 表现层（Web API）

- 控制器只做参数接收与返回，业务逻辑下沉。
- 所有接口异步实现。
- 认证与授权（RBAC + 最小权限）。

### 3.5 模块关键能力

**员工管理**
- 账号生命周期管理（创建/启停/状态流转）。
- 登录标识（用户名/手机号/邮箱）维护。
- 密码策略、首次登录改密、重置流程。
- 角色/权限分配、部门与职位绑定。
- 审计日志（创建、启停、授权、重置等）。

**部门管理**
- 树形结构维护、排序与上下级调整。
- 组织编码唯一性校验。
- 组织维度统计查询。

**职位管理**
- 岗位信息维护与启停。
- 主岗/兼岗关联规则。
- 基于岗位的权限模板（可选）。

**租户管理**
- 租户创建、启停、管理员设置。
- 租户资源配额（可选）。
- 租户级数据隔离与安全访问控制。

**应用管理**
- 应用生命周期管理（启停、配置、发布状态）。
- 租户开通/变更/停用流程与审计。

**应用数据源管理**
- 数据源维护与连接测试。
- 应用-数据源绑定策略。
- 变更记录、审批/窗口控制（可选）。
- 连接信息安全存储与脱敏展示。
